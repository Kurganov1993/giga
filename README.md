Всем привет! Сегодня я хочу поделиться опытом создания веб-приложения на основе GigaChat API от Сбера. В проекте я использовал не только текстовый диалог с нейросетью, но и добавил голосовой ввод (распознавание речи) и озвучку ответов с помощью SaluteSpeech. Получился полноценный голосовой AI-ассистент. В этой статье я расскажу о технических деталях: как получить доступ к API, как организовать обмен сообщениями, кэшировать токены, обрабатывать ошибки и сделать удобный интерфейс.

Почему GigaChat?
На рынке много языковых моделей: ChatGPT, Claude и другие. Но GigaChat привлек меня несколькими моментами:

Это российская разработка, которая хорошо понимает русский язык и культурный контекст.

На момент разработки был доступен бесплатный тариф с достаточными лимитами для тестирования.

Поддержка различных режимов: от простых вопросов до генерации кода.

Интеграция с экосистемой Сбера (например, SaluteSpeech для синтеза речи).

Архитектура проекта
Я выбрал Next.js (App Router) для создания full-stack приложения. На серверной части реализованы API routes для безопасного взаимодействия с GigaChat и SaluteSpeech. Клиентская часть написана на React с TypeScript, используется Tailwind CSS для стилей.

Основные компоненты:

Серверный эндпоинт для GigaChat — получает сообщение от клиента, валидирует, получает токен доступа, отправляет запрос к API и возвращает ответ.

Серверный эндпоинт для SaluteSpeech — аналогично, для синтеза речи (я использовал его через хук useSaluteSpeech).

Клиентский интерфейс — список чатов, поле ввода, отображение сообщений с возможностью редактирования и оценки.
